Status: **canonical evidence classification as of 2026-07-12**.

This document governs interpretation of every earlier POC report, metrics file,
gate result, and case-study verdict in this repository. Raw artifacts are
retained for auditability; contradictory or weak evidence is labeled rather
than deleted.

## Current Verdict

**No existing benchmark establishes the existence thesis in the current
[Convergent Programming Concept](../docs/SCULPT_Convergent_Programming_Concept.md).**

The current repository has useful prototype and engineering evidence. It has no
decision-grade evidence that a typed SCULPT semantic/evidence graph is better
than best-practice spec-driven agentic development.

All earlier Go, No-Go, PASS, FAIL, and release-gate labels are historical
results under their original local protocols. They are superseded for strategic
decision-making by this classification.

## Explicit Corrections

1. The Incident Triage claim of a clear reproducibility win is withdrawn as an
   existence claim. File-hash stability from a deterministic path is not a fair
   comparison with repeated free regeneration.
2. The data-heavy benchmark is **unresolved**, neither a clean SCULPT loss nor a
   valid synthesis win. The 2026-03-03 report is stale relative to later
   deterministic fixes, while later PASS artifacts test compiler-owned
   deterministic behavior rather than the intended agentic system.
3. Byte-identical or normalized-identical output is not evidence of AI
   convergence when both arms ultimately execute equivalent hand-written
   business logic.
4. Stub and replay providers may test infrastructure, but they cannot count as
   evidence for real synthesis.
5. Existing release gates protect prototype behavior only. Their names do not
   confer decision-grade benchmark status.

## What Remains Useful

- parser, semantic, contract, and diagnostic regression coverage;
- benchmark harness and artifact-capture lessons;
- datasets and acceptance scenarios as possible non-holdout development
  fixtures;
- explicit records of infrastructure failures and fallbacks;
- evidence that deterministic code paths can be repeatable;
- evidence that current benchmark design can produce misleading claims.

None of these may be reused as a holdout result in the future existence
experiment.

## Requirements For The Next Valid Claim

No new competitive claim is allowed until the concept's commercial, language,
loop, graph, operations, and provider gates pass.

The decisive experiment must:

- compare Full SCULPT, the same harness without the typed graph, and a
  best-practice spec-driven native coding agent;
- use the same model snapshot, repository, tools, permissions, tests, budgets,
  and repair rights;
- freeze providers and protocol before holdout task selection;
- use at least four external Brownfield repositories from two organizations in
  a provider corridor frozen before commercial interviews;
- treat each interacting repository change sequence as an inference cluster,
  with cluster-aware power and analysis;
- select the best-practice baseline through an independent, equal-budget
  non-holdout bake-off before holdout tasks exist;
- include hidden tests and independent review;
- measure fully loaded cost and human assurance effort;
- retain all raw data, failed attempts, fallbacks, and interventions;
- measure semantic and obligation reproducibility rather than file hashes alone.

Until that experiment passes every immutable threshold, the only honest project
status is: **coherent hypothesis, unproven product**.
